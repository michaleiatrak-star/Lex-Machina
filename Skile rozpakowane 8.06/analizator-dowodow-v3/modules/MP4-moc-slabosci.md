# M4 — Mocne i słabe strony: sąd, przeciwnik, własny pełnomocnik

## Cel
Ocenić materiał nie deklaratywnie, lecz procesowo: co sąd uzna, co przeciwnik podważy, co trzeba naprawić.

## 4.1 Ocena per dziedzina

Dla każdej dziedziny z M2:

```text
DZIEDZINA:
Kluczowy standard prawny:
Element decydujący:
Fakty wspierające:
Dowody:
Luki:
```

## 4.2 Perspektywa sędziego

```text
Co sąd prawdopodobnie uzna za udowodnione:
Czego sąd nie uzna bez dodatkowego dowodu:
Co sąd uzna za prawnie irrelewantne:
Jak sąd oceni ciężar dowodu:
Najważniejsze pytanie sądu:
Najbardziej prawdopodobne ustalenie:
```

## 4.3 Perspektywa przeciwnika

```text
Najmocniejszy atak przeciwnika:
Atak na fakty:
Atak na dowody:
Atak na związek przyczynowy:
Atak na wysokość roszczenia:
Atak formalny / terminowy:
Atak na wiarygodność:
Szansa powodzenia ataku: 0–100%
```

## 4.4 Perspektywa własnego pełnomocnika

```text
Co eksponować jako pierwsze:
Co trzeba udowodnić dodatkowo:
Jak zabezpieczyć lukę:
Jak odpowiedzieć na najmocniejszy kontrargument:
Czego nie akcentować, bo szkodzi:
Rekomendacja procesowa:
```

## 4.5 Skala elementów

- `💪💪💪` — bardzo mocne: dokument + spójność + relewantność prawna.
- `💪💪` — mocne: dowód jest, ale wymaga interpretacji.
- `💪` — umiarkowane: logiczne, lecz podatne na kontrę.
- `⚠️` — słabość umiarkowana: luka możliwa do naprawy.
- `⚠️⚠️` — słabość poważna: przeciwnik realnie wykorzysta.
- `⚠️⚠️⚠️` — krytyczna: może przesądzić o oddaleniu / przegranej.

## 4.6 Element podważalny

```text
[E-POD001]
Element:
Dlaczego podatny:
Jak zaatakuje przeciwnik:
Jak odpowie sąd, jeśli brak riposty:
Riposta:
Dowody do uzupełnienia:
Priorytet: pilne / ważne / pomocnicze
```

## 4.7 Predykcja procesowa

```text
Roszczenie / zarzut:
Szansa uwzględnienia: wysoka / średnia / niska / nie do oceny
Warunek sukcesu:
Warunek porażki:
Najbardziej prawdopodobne rozstrzygnięcie:
Uzasadnienie jak sąd:
Zastrzeżenia:
```

## 4.8 Zakaz pozornej symetrii

Nie twórz sztucznej równowagi. Jeżeli jedna strona ma silniejszy materiał, wskaż to wprost. Jeżeli użytkownik ma słaby punkt, nazwij go bez łagodzenia.
