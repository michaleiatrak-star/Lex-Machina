# MATRIX-COMPLETENESS-AUDIT-GATE

## Cel
Kontrola kompletności matryc.

## Minimalne pola każdej matrycy
Każda matryca powinna mieć co najmniej:
- nazwę jednostki albo źródła,
- kategorię,
- organ nadrzędny/nadzorczy,
- podstawę ustawową,
- publikator,
- status pokrycia,
- brakujące warstwy,
- datę ostatniej weryfikacji,
- status źródłowy.

## Statusy kompletności
- `COMPLETE`
- `PARTIAL`
- `MISSING_SUBMODULES`
- `REQUIRES_SOURCE_INGESTION`
- `REQUIRES_ISAP_CHECK`
- `REQUIRES_BIP_CHECK`
- `REQUIRES_DUW_CHECK`
- `REQUIRES_REGISTRY_CHECK`
