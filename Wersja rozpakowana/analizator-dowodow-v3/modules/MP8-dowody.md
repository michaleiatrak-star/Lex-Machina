# M8 — Matryca dowodowa i ocena mocy dowodów

## Cel
Przypisać dowody do faktów i przesłanek prawnych, ocenić ich moc, legalność, kompletność i podatność na atak.

## 8.1 Rejestr dowodów

```text
[DOW001]
Nazwa dowodu:
Typ: dokument / e-mail / SMS / nagranie / świadek / opinia / protokół / zdjęcie / metadane / inny
Kto dostarcza:
Jaki fakt dowodzi:
Źródło:
Oryginał czy kopia:
Czy załączony:
Czy wymaga uwierzytelnienia:
Moc: wysoka / średnia / niska
Ryzyko podważenia:
```

## 8.2 Matryca fakt–dowód–przesłanka

```text
Fakt ID | Przesłanka prawna | Dowód ID | Moc dowodu | Luka | Działanie
```

## 8.3 Ocena admissibility / dopuszczalności

Sprawdź:

- czy dowód jest legalny;
- czy został zgłoszony w terminie;
- czy jest relewantny;
- czy nie jest spóźniony;
- czy wymaga oryginału;
- czy wymaga biegłego;
- czy narusza tajemnicę, prywatność albo dane osobowe;
- czy przeciwnik może żądać pominięcia.

## 8.4 Chain of custody

Dla dowodów cyfrowych i nagrań:

```text
Kto utworzył:
Kiedy utworzył:
Gdzie przechowywano:
Czy są metadane:
Czy była edycja:
Czy jest kopia źródłowa:
Czy można potwierdzić autentyczność:
Jak zabezpieczyć:
```

## 8.5 Dowody negatywne i braki

Brak dowodu też jest informacją:

```text
Brakujący dowód:
Kto powinien go mieć:
Dlaczego powinien istnieć:
Jak żądać przedstawienia:
Skutek procesowy braku:
```
