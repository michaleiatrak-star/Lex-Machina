# M0 — Intake, mapa materiału i ograniczenia analizy

## Cel
Ustalić, co dokładnie zostało dostarczone, jaki jest status dokumentów, czego brakuje i jakie moduły należy uruchomić. M0 chroni przed analizą opartą na niepełnym materiale.

## 0.1 Rejestr dokumentów

Dla każdego dokumentu utwórz kartę:

```text
DOC-ID: DOC01
Nazwa:
Typ:
Autor / strona:
Adresat:
Data dokumentu:
Data doręczenia / złożenia:
Sygnatura:
Sąd / organ:
Liczba stron:
Załączniki wymienione:
Załączniki faktycznie dostarczone:
Status: kompletny / fragment / skan / transkrypcja / brak załączników
Ryzyko techniczne: niskie / średnie / wysokie
```

## 0.2 Klasyfikacja sprawy

Oznacz dominujące obszary:

- cywilne;
- pracownicze;
- karne;
- administracyjne;
- dyscyplinarne;
- RODO / cyber;
- gospodarcze;
- rodzinne / spadkowe;
- inne.

## 0.3 Hipoteza robocza sprawy

Nie jest to ustalenie faktu. To plan analizy.

```text
Główna oś sporu:
Roszczenia / zarzuty:
Najważniejsze fakty do udowodnienia:
Najważniejsze dowody:
Największe luki już widoczne:
Tryb modułów:
```

## 0.4 Ograniczenia

Wskaż jawnie:

- brakujące strony;
- brakujące załączniki;
- nieczytelne skany;
- brak dat;
- brak podpisów;
- brak metadanych;
- rozbieżności nazewnictwa plików;
- brak możliwości weryfikacji nagrań / obrazów / oryginałów.

## 0.5 Decyzja routingowa

Zapisz, które moduły uruchomić i dlaczego. Jeżeli pomijasz M3, M5 lub M6, podaj powód.
