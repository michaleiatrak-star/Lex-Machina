# POST-VALIDATION — stub (sub-referencja)

Pełna logika walidacji spójności po wygenerowaniu pisma (oferta walidacji,
cztery warstwy analizy, klasyfikacja braków, aktualizacja pisma) znajduje się
w pliku głównym.

**Wczytaj przed użyciem:**
```
view /mnt/skills/user/shared/POST-VALIDATION.md
```

Uruchom zgodnie z instrukcją tamtego pliku — wszystkie fazy (1–5), format raportu
i sekwencja aktualizacji są tam zdefiniowane.

**Skrót dla tego skilla:**
Po wygenerowaniu pisma → wczytaj plik główny → zaoferuj walidację (Faza 1) →
jeśli TAK: przeprowadź analizę warstw A–D → wygeneruj raport braków → zaktualizuj
pismo na podstawie odpowiedzi użytkownika.
