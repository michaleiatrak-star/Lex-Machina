# WITNESS-INTELLIGENCE — Profil Wiedzy i Dokumentacja Świadka

> Moduł fazy przygotowawczej (pre-W1). Wykonywany PRZED etapem intake W1.
> Cel: zbudować kompletną mapę:
>   (1) kim jest świadek — profil osobowy i procesowy,
>   (2) co świadek MOŻE wiedzieć — zakres wiedzy z dokumentów,
>   (3) co wiemy o świadku z akt — cytaty, fakty, powiązania,
>   (4) mapa źródeł per temat — każdy fakt z dok_id + strona.
>
> Źródło metodyki: Wagner (Taft Law 2024) — "Get Everything You Can About
> The Witness"; Pozner & Dodd — "Cross Preparation Systems: Sourcing the Facts";
> Filevine (2024) — "Thoroughly Review All Case Materials".

---

## FAZA PRE-W1 — SEKWENCJA OBLIGATORYJNA

```
KROK I  — PROFIL ŚWIADKA (kim jest)
KROK II — MAPA WIEDZY (co może wiedzieć)
KROK III — EKSTRAKCJA DOKUMENTACYJNA (co wiemy o świadku z akt)
KROK IV — MAPA ŹRÓDEŁ PER TEMAT (preparation chart)
KROK V  — OCENA WSTĘPNA (zasilenie W1/W2)
```

Każdy krok wykonaj z dostarczonych materiałów. Pytaj użytkownika TYLKO
o luki których dokumenty nie wypełniają.

## KROK I — PROFIL ŚWIADKA

### I-A. Dane podstawowe

Ekstrahuj z akt lub pytaj:

| Pole | Źródło | Priorytet |
|---|---|---|
| Imię / inicjały / oznaczenie anonimowe | akta, nagłówek protokołu | krytyczny |
| Wiek / rok urodzenia | akta, protokół | uzupełniający |
| Wykształcenie / zawód | akta, zeznania wcześniejsze | istotny |
| Stanowisko w dacie zdarzenia | akta HR, umowy, org-chart | krytyczny |
| Obecne zatrudnienie / status | akta, zeznania | istotny |
| Narodowość / język zeznań | protokół | krytyczny jeśli niepolski |

### I-B. Profil relacyjny

```
Pytania do ekstrakcji z dokumentów:
□ Jaki jest stosunek świadka do stron postępowania?
  → bezpośredni pracodawca / podwładny / współpracownik / kontrahent /
    osoba trzecia / biegły / świadek ze zdarzenia
□ Czy świadek ma interes w wyniku sprawy (finansowy, reputacyjny, karny)?
□ Czy świadek jest powiązany personalnie z którąś ze stron?
□ Jak zmieniła się relacja ze stronami po zdarzeniu/sporze?
□ Czy świadek złożył oświadczenia pisemne przed postępowaniem (e-mail,
  notatka, protokół wewnętrzny, pismo do PIP/ZUS/innego organu)?
```

### I-C. Historia w postępowaniu

```
□ Czy świadek był przesłuchiwany wcześniej w tej sprawie?
  → jeśli TAK: oznacz daty i sygnatury protokołów → PRIOR-TESTIMONY-GATE
□ Czy świadek zeznawał w powiązanych postępowaniach (cywilnym/karnym/insp.)?
□ Czy złożył zeznania dla organu (PIP, ZUS, prokuratura)?
□ Czy przygotował pisemne oświadczenia procesowe?
□ Czy był cytowany w pismach strony przeciwnej?
```

### I-D. Profil zachowania (jeśli dostępne)

```
Źródło: wcześniejsze protokoły przesłuchań, opisy w pismach procesowych.
□ Jak świadek zachowywał się podczas poprzednich zeznań?
  → kooperatywny / ostrożny / unikający / konfrontacyjny / popadający w sprzeczności
□ Czy stosował formułę "nie pamiętam" / "nie wiem"?
□ Czy odpowiadał poza zakresem pytania?
□ Czy zeznania były spójne wewnętrznie?
```

## KROK II — MAPA WIEDZY ŚWIADKA

> Cel: określić czego świadek mógł się dowiedzieć na podstawie jego roli
> i dostępu do informacji — NIEZALEŻNIE od tego co zeznał.

### II-A. Zakres wiedzy wynikający z roli

```
Na podstawie KROKU I (stanowisko, rola, czas) zbuduj mapę:

WIEDZA BEZPOŚREDNIA (świadek był uczestnikiem):
  → lista zdarzeń/decyzji, przy których świadek był obecny lub które
    współtworzył (z odwołaniem do dokumentów)

WIEDZA PROCEDURALNA (świadek miał dostęp z racji stanowiska):
  → dokumenty, które powinien był widzieć/podpisać/zatwierdzić
  → systemy/bazy danych do których miał dostęp
  → osoby z którymi regularnie się komunikował o sprawach spornych

WIEDZA ZE SŁYSZENIA (świadek mógł się dowiedzieć pośrednio):
  → od kogo / kiedy / w jakich okolicznościach mógł uzyskać informacje
  → spotkania, narady, e-maile CC/BCC, raporty

WIEDZA WYKLUCZONA (czego świadek nie mógł wiedzieć):
  → decyzje podjęte przed/po jego zatrudnieniu
  → dokumenty do których nie miał dostępu
  → zdarzenia przy których nie był obecny
  → ważne: zaznacz WIEDZĘ WYKLUCZONĄ bo to obszar "nie wiem" świadka
    → te obszary NIE nadają się do pytań otwartych
```

### II-B. Luki wiedzy = pułapki pytań

```
REGUŁA: pytanie o fakt z obszaru WIEDZY WYKLUCZONEJ lub WIEDZY ZE SŁYSZENIA
bez wskazania konkretnego kanału informacji → RYZYKO-KONTROLA (świadek
odpowie "nie wiem" lub rozszerzy narrację).

Dla każdego tematu przesłuchania oznacz:
  [BEZP]   — wiedza bezpośrednia → pytania zamknięte, ONE-FACT
  [PROC]   — wiedza proceduralna → pytania o dokument + dostęp
  [SŁYSZ]  — wiedza ze słyszenia → pytania o kanał + czas przekazu
  [WYKLUCZ]— wiedza wykluczona  → NIE pytaj lub pytaj tylko o potwierdzenie
                                   BRAKU wiedzy (taktyczne)
```

## KROK III — EKSTRAKCJA DOKUMENTACYJNA

> Cel: zebrać wszystko co akta mówią o świadku i przez świadka.
> Metodyka: Wagner 2024 — "Do you have any exhibits the witness prepared /
> sent to / received by / should have seen?"

### III-A. Dokumenty AUTORSTWA świadka

```
Przeskanuj dostarczone materiały i wypełnij tabelę:

| dok_id | Typ dokumentu | Data | Treść kluczowa (1 zdanie) | Strona/zakładka |
|---|---|---|---|---|
```

Typy do szukania: e-mail, pismo wewnętrzne, notatka służbowa, protokół,
raport, wniosek, umowa (podpisana przez świadka), oświadczenie pisemne,
wiadomość SMS/chat.

### III-B. Dokumenty SKIEROWANE DO świadka

```
| dok_id | Typ | Od kogo | Data | Treść kluczowa | Strona/zakładka |
|---|---|---|---|---|---|
```

### III-C. Dokumenty KOPIOWANE świadkowi (CC/BCC) lub w których jest wzmiankowany

```
| dok_id | Typ | Kontekst wzmianki | Data | Strona/zakładka |
|---|---|---|---|---|
```

### III-D. Cytaty i twierdzenia świadka z wcześniejszych zeznań

```
Dla każdego protokołu przesłuchania / oświadczenia:

PROTOKÓŁ: [sygnatura / data / oznaczenie]
─────────────────────────────────────────
CYTAT-01: "[dosłowne sformułowanie świadka]"
           Kontekst: [pytanie lub okoliczność]
           Strona: [X]
           Kwalifikacja: [fakt bezspory / twierdzenie sporne / niejasność]

CYTAT-02: ...
```

### III-E. Fakty dotyczące świadka cytowane przez INNE źródła

```
Jeśli inne zeznania lub dokumenty powołują się na świadka:

| Źródło | Co twierdzi o świadku | Zgodność z zeznaniami świadka |
|---|---|---|
```

## KROK IV — MAPA ŹRÓDEŁ PER TEMAT (Preparation Chart)

> Cel: grupowanie faktów tematycznie z pełną ścieżką źródłową.
> Każdy temat = jedna strona/sekcja. Metodyka: Pozner & Dodd 4th ed.
> (2024) — "Cross Preparation Systems: Sourcing the Facts".

### Format mapy

```
═══════════════════════════════════════════════════════
TEMAT: [nazwa tematu — wąsko i precyzyjnie]
Cel procesowy: [co chcę udowodnić tym tematem]
Teza: [T_X z W2]
Wiedza świadka: [BEZP / PROC / SŁYSZ / WYKLUCZ]
═══════════════════════════════════════════════════════

FAKTY WSPIERAJĄCE TEZĘ:
┌─────────────────────────────────────────────────────┐
│ FAKT-01: [treść faktu jednym zdaniem]               │
│ Źródło:  dok_id=[X], strona=[Y], typ=[e-mail/protokół/umowa/...] │
│ Cytat:   "[dosłowne sformułowanie jeśli dostępne]"  │
│ Siła:    [mocny / umiarkowany / słaby]              │
│ Pytanie: [propozycja pytania zamkniętego ONE-FACT]  │
└─────────────────────────────────────────────────────┘
┌─────────────────────────────────────────────────────┐
│ FAKT-02: ...                                        │
└─────────────────────────────────────────────────────┘

FAKTY ZAGROŻONE (świadek może próbować im zaprzeczyć):
┌─────────────────────────────────────────────────────┐
│ ZAGROŻENIE-01: [co świadek może powiedzieć]         │
│ Kontrowanie:   [dokument lub fakt zamykający ucieczkę] │
│ Źródło kontrowania: dok_id=[X], strona=[Y]          │
└─────────────────────────────────────────────────────┘

SPRZECZNOŚCI (jeśli dostępne wcześniejsze zeznania):
┌─────────────────────────────────────────────────────┐
│ SPRZECZNOŚĆ-01:                                     │
│   Wcześniej (protokół [data], str. [X]):            │
│     "[cytat]"                                       │
│   Teraz (spodziewane zeznanie lub twierdzenie):     │
│     [opis]                                          │
│   Sekwencja 3-pytań (BLOK D):                       │
│     Q1: [potwierdzenie aktualnej wersji]            │
│     Q2: [konfrontacja z wcześniejszym]              │
│     Q3: [pytanie o przyczynę rozbieżności — opcj.] │
└─────────────────────────────────────────────────────┘
```

### Liczba tematów

```
Buduj preparation chart dla KAŻDEGO tematu/rozdziału planowanego W3.
Eksperci (Wagner 2024): przygotuj 8-10 tematów, do przesłuchania użyj 4-6.
Nadwyżka tematów = rezerwa jeśli przesłuchanie pójdzie inaczej niż planowano.
```

## INTEGRACJA Z PIPELINE W1→W2→W3

```
KROK V → W1 INTAKE:
  Wypełnia automatycznie:
    WARSTWA A: dane świadka z KROK I
    WARSTWA B (typ 1): stanowisko, dostęp, stosunek do stron z KROK I-B
    PRIOR-TESTIMONY-GATE: dane z KROK I-C
    INTAKE-COMPLETENESS-GATE: pre-weryfikacja z KROK V

KROK IV → W2 TEZY:
  Tematy z preparation chart → tezy główne (Priorytet A)
  Sprzeczności → tezy do obalenia + materiał BLOKU D
  Wiedza wykluczona → ostrzeżenia BLOK E

KROK III+IV → W3 PYTANIA:
  FAKT-XX + dok_id + strona → FPW-1 każdego pytania (gotowa ścieżka źródłowa)
  Sekwencja 3-pytań ze SPRZECZNOŚCI → BLOK D (gotowa)
  ZAGROŻENIE-XX + kontrowanie → dowód kontrolny dla pytania
```
