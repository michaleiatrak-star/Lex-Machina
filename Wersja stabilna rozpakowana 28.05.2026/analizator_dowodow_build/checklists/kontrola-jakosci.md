# Checklist — kontrola jakości analizy

## Podstawowe wymagania (zawsze)

- [ ] Każdy fakt ma źródło (ID z M1).
- [ ] Każdy cytat jest dosłowny.
- [ ] Fakty oddzielono od interpretacji.
- [ ] Oznaczono braki dokumentów.
- [ ] Oznaczono kolizje.
- [ ] Wskazano ciężar dowodu.
- [ ] Wskazano najmocniejszy argument przeciwnika.
- [ ] Wskazano słabe punkty użytkownika.
- [ ] Wskazano realne ryzyka procesowe.
- [ ] Przepisy do aktualnej weryfikacji oznaczono (ISAP).
- [ ] Orzecznictwo nie jest generowane bez sprawdzenia (orzeczenia-sadowe-v2).
- [ ] Rekomendacje są operacyjne.
- [ ] Terminy krytyczne (< 14 dni) oznaczono alertem M12.

## Synteza faktyczna M13 (gdy M13 uruchomiony)

- [ ] Każdy łańcuch faktyczny zakorzeniony w ID z M1 — brak wiszących ogniw.
- [ ] Każdy łańcuch ma wskazane najsłabsze ogniwo i ripostę.
- [ ] Zbieżności zawierają alternatywną interpretację przeciwnika.
- [ ] Narracja procesowa istnieje w wersji własnej ORAZ wersji przeciwnika.
- [ ] Fakty szkodliwe mają przypisaną strategię neutralizacji.
- [ ] Fakty [RYZYKO-NARRACYJNE] bez neutralizacji = defekt krytyczny (zablokuj eksport).
- [ ] Test spójności narracyjnej (M13 sekcja 13.7) przeprowadzony; wynik zapisany w M9.
- [ ] Blok [BLOK-M13→M7] skonsumowany przez M7 — sekcje 7 i 8 raportu wypełnione.
- [ ] Powiązania bez źródła oznaczone [H-ŁAŃCUCH], nie prezentowane jako fakty.
- [ ] Zbieżność słabych faktów opatrzona jawnym zastrzeżeniem o sile argumentacyjnej.
