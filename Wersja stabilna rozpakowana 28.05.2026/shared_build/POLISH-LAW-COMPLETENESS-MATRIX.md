# POLISH-LAW-COMPLETENESS-MATRIX — mapa kompletności prawa polskiego

## Status
Ten plik jest rejestrem jakościowym. Nie jest źródłem brzmienia prawa. Brzmienie prawa i Dz.U. weryfikuj w ISAP.

## Poziomy dojrzałości modułu

| Poziom | Znaczenie |
|---|---|
| A | moduł wzorcowy: intake + workflow + dowody + ryzyka + strategia + ISAP + quality gate |
| B | moduł operacyjny: ma procedurę i dowody, ale wymaga pogłębienia strategii albo orzecznictwa |
| C | moduł opisowy: wymaga uzupełnienia workflow |
| X | brak modułu albo moduł nieaktywny |

## Wymóg aktualny
Wszystkie moduły prawa polskiego po tej aktualizacji muszą osiągać co najmniej poziom A-funkcjonalny przez:

- własne sekcje szczególne,
- import `MODULE-STANDARD-POLISH-LAW`,
- import shared: `FORMAL-CHECK`, `WARUNKI-SKUTECZNOSCI`, `RISK-ASSESSMENT`, `DOWODY-METODOLOGIA`, `TEMPORAL-LAW-CHECK`.

## Bramka kompletności
Moduł jest kompletny dopiero wtedy, gdy odpowiada na pytania:

- jaki organ/sąd jest właściwy,
- jaki tryb stosujemy,
- jaki termin i od kiedy biegnie,
- jakie pismo/środek jest skuteczny,
- jakie dowody są decydujące,
- kto ma ciężar dowodu,
- jakie są ryzyka formalne,
- jakie są ryzyka materialne,
- jaka jest strategia procesowa,
- jak wygląda ścieżka odwoławcza,
- co trzeba sprawdzić w ISAP przed cytowaniem.
